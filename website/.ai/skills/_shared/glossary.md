# Skill glossary

Terms used across skills. For project-wide terms see [`../../context/glossary.md`](../../context/glossary.md).

| Term | Meaning |
| --- | --- |
| Acceptance criteria | Observable, independently checkable outcomes listed in a task file. |
| Blocker | A finding severe enough to stop merge. See [`../../checklists/pull-request.md`](../../checklists/pull-request.md). |
| Regression test | A test that would have failed on the pre-fix code path. |
| Root cause | The deepest defect whose removal resolves the symptom without re-introduction. |
| Smallest correct change | The minimal diff that satisfies the acceptance criteria and rules. |
